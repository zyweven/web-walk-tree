## 目标
- 用成熟的前端图形库替换当前自绘 SVG，提供稳定、可交互的“思维导图/树图/径向”视图。
- 支持平移/缩放、节点点击打开、层级布局、标签友好显示，适配大量节点。

## 组件候选与对比
- Cytoscape.js（MIT）：
  - 优点：稳定、性能好、交互完善（平移/缩放/选中/样式/事件）。
  - 布局：内置 `breadthfirst`（树/思维导图）、`concentric`（径向）；配套 `cytoscape-dagre`（分层有向图），`cytoscape-cola`（力导向）。
  - MV3 兼容：纯前端，无 eval 需求，可本地引入。
- vis-network（MIT）：
  - 优点：简单易用，层级/力导向、物理引擎、交互完整。
  - 风格偏网络图，思维导图效果可达，但标签控制略弱。
- jsMind（MIT）：
  - 优点：专注思维导图（树），标签显示友好。
  - 局限：DAG/多父不自然；适合“树视图”但不适合“时间线/径向”。
- Elkjs + 手绘：
  - 优点：布局高质量（分层/正交），可控。
  - 成本：需自研渲染与交互，不适合快速稳定交付。

结论：优先选 Cytoscape.js + `breadthfirst`（思维导图）+ `concentric`（径向）+ 可选 `cytoscape-dagre`（分层）。保留备选 vis-network 以防万一。

## 集成方案（Manifest V3 无网络依赖）
1) 在 `extension/sidepanel/lib/` 引入：
   - `cytoscape.min.js`
   - `cytoscape-dagre.min.js`（可选）
2) 在 `sidepanel/index.html` 以 `<script src>` 本地引用库。
3) 在 `sidepanel/index.js` 新增 `renderMindmapWithCytoscape` 与 `renderRadialWithCytoscape`：
   - 数据映射：
     - nodes → `{ data: { id, label, url, domain, isRoot } }`
     - edges → `{ data: { id, source: fromNodeId, target: toNodeId } }`
     - 仅保留 `http/https`，并基于当前标签溯源确定根。
   - 思维导图布局：
     - `layout: { name: 'breadthfirst', directed: true, animate: false, spacingFactor: 1.2, circle: false, roots: [rootId] }`
     - 文本多行：使用 `
` 处理，或通过 `cy-node-html-label`（若无需额外插件则先用内置标签）。
   - 径向布局：`concentric` 并按深度控制同心层。
   - 样式：节点圆点/根节点高亮、边淡色、标签限制宽度、鼠标悬浮显示完整标题。
   - 交互：
     - 点击节点：`cy.on('tap','node', open url)`。
     - 平移缩放：默认开启，设置 `minZoom/maxZoom` 防止溢出。
4) 视图切换与偏好：沿用 `sidepanelView` 持久化；默认“思维导图”。

## 性能与质量保证
- 过滤与限量：每层限制节点数（如 50），提供“更多...”提示。
- 大图优化：禁用动画，延迟批量 `cy.add`，按域名聚合（后续可做折叠）。
- 文案与标签：域名 + 断行标题，避免溢出；容器自适应。

## 回退机制
- 若 Cytoscape 在特定页面出现兼容问题，保留“时间线/树（日常）”视图并可切回。
- 必要时引入 vis-network 的 `hierarchical` 作为备用实现。
